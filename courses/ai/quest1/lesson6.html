🔄 Transfer Learning

Transfer learning lets you take a model trained on a large dataset (like MobileNet on ImageNet) and reuse it for a new, similar task. You only need to retrain the final layer on your own data – saving time and computational power.

"Think of it as learning to ride a bicycle – once you know balance, you can quickly adapt to a motorcycle. The core skills transfer."

🧠 How Transfer Learning Works

  • Pre-trained model – MobileNet (trained on 1000+ object categories).
  • Feature extraction – Use the model as a fixed feature extractor (remove the top classification layer).
  • New classifier – Add a new dense layer (or a few layers) and train it on your dataset.
  • Fine-tuning (optional) – Unfreeze some layers and continue training with a low learning rate.
MobileNet (frozen) → Embeddings (1024 dims) → New Dense(2) → Softmax
transfer_learning.js
// Load pre-trained MobileNet
const mobilenet = await mobilenet.load();

// Extract embeddings for each image
const embedding = mobilenet.infer(img, true); // true = embeddings

// Build new classifier on top
const newModel = tf.sequential();
newModel.add(tf.layers.dense({units: 128, activation: 'relu', inputShape: [1024]}));
newModel.add(tf.layers.dense({units: 2, activation: 'softmax'}));

🌍 Transfer Learning in Practice

  • 🎨 Style transfer (e.g., Prisma app)
  • 🖼️ Custom image classifiers with small datasets
  • 🗣️ Speech recognition (reuse acoustic models)
  • 📄 Text classification (BERT fine‑tuning)
  • ⚕️ Medical imaging (X‑ray, MRI) with limited labelled data
